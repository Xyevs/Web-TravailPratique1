Intrduction

Introduction Vous avez été engagé par le mercenaire Bane pour lui écrire un langage de programmation écrit en C++ pour l’aider dans sa lutte contre le Chevalier Noir. Évidemment, une récompense substantielle vous sera remise à la livraison du produit. Le travail sera séparé en deux parties (le TP2 et le TP3). Les deux parties vont fonctionner ensemble ! Ainsi, vous ne pouvez pas vraiment vous permettre de complètement « rater » la première partie.

Compilation

Lorsqu’un programmeur du langage Bane voudra écrire une application, il va écrire dans un fichier texte brut (avec l’extension naturelle « .bane », mais ce n’est pas obligatoire) tout simplement les lignes de code à compiler. Votre compilateur « Bane » doit prendre comme premier argument un fichier à compiler écrit dans le langage Bane. Il peut prendre également d’autres arguments :

  • « -c » : Permet de créer un fichier C++ qui correspond à la translation du code Bane en code C++ dans le répertoire courant qui porte le même nom que le fichier d’entré dans le langage Bane.
  • « -e » : Permet de nommer l’environnement d’exécution de l’application Bane. Si ce paramètre n’est pas spécifié, l’environnement se nommer automatiquement Gotham. Il s’agit également du nom de l’exécutable sur le disque.

Exemple de compilation sans argument. L’environnement d’exécution sera « Gotham » :

> bane test.bane –c –e BreakTheBat


Environnement d'exécution

Un programme écrit dans la langage Bane n’est pas destiné à être « exécuter » comme un programme applicatif standard. En Bane, il s’agit plutôt de programmer un environnement d’exécution destiné à l’écriture de pseudo-requêtes qui feront appel aux fonctionnalités que le programmeur Bane aura écrit dans son code. Ainsi, lorsqu’on compile un programme Bane, on obtient un programme exécutable qui représente un environnement d’exécution qui fonctionne exactement de la même façon peut importe le code Bane qui a été compiler. Voici un exemple de compilation et ensuite d’exécution :


> bane test.bane –e BreakTheBat
> ls test.bane BreakTheBat > BreakTheBat
